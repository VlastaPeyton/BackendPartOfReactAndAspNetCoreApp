Unit test:
	Proveri logiku.
	Testira klasu/njene metode dok sve DI oko nje je simulirano (mokovano).
	Pokriva Service/CQRS, Helpers, Validator. 

Integration test:
	Testiram prave komponente (repository , controller ), ne mokovane. Tj sve sto zavisi od baze i API.
	Koristi test bazu.
	Koristi EF Core, DbContext za razliku od Unit test.
	Proveri da li slojevi saradjuju kako treba.

Moj app sadrzi Middleware -> Controller i Endpoint -> Service/CQRS -> Repository -> DbContext -> Db 

Unit test izgledace:
	NuGet packages: xUnit, Moq/FakeItEasy, FluentAssertions 
	Arrange: mock repository, service/cqrs handler sa tim mock, input podatke
	Act: pozoves metodu
	Assert: provera rezultata

		Provera da li mapping radi
